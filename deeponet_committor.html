DeepONet for Parametric Committor Learning

This project focuses on neural-operator learning for a one-dimensional committor equation. The target is not a single PDE solution, but the operator that maps an entire family of potential functions to the associated committor. I started from the original DeepONet setup on Gaussian random fields, found that its generalization was weak on unseen potentials, and then redesigned the data representation and architecture around a cleaner parametric problem.

Main role: redesign a DeepONet pipeline for operator learning on a parametric committor family, improving test RMSE from roughly \(8.9\times10^{-2}\) in the original GRF-based setting to \(6.68\times10^{-3}\) on unseen potentials in the structured parametric setting.

Problem Definition

The committor problem is posed on the interval \([-1,1]\):

\[ q''(x)-\beta V'(x)q'(x)=0, \qquad q(-1)=0,\; q(1)=1, \qquad \beta=1. \]

The goal is to learn an operator that maps the potential derivative or its low-dimensional parameterization to the committor function. Ground-truth labels are generated with a Chebyshev spectral solver implemented in Committor1D_helpers.py, so the learned model is always evaluated against a high-quality numerical reference.

Why the Original DeepONet Setup Was Not Enough

The initial DeepONet experiment followed the standard function-to-point setup using Gaussian random fields as inputs. In that formulation, 100 GRF samples were discretized on 100 spatial points, and each committor was queried at 100 random points, leading to 10,000 training tuples of the form

\[ [u(x_0),u(x_1),\dots,u(x_{N_x-1}),y,q(y)]. \]

This baseline trained successfully but generalized poorly. The training MSE was approximately \(1.36\times10^{-5}\), yet the test MSE on unseen GRF samples was about \(3.37\times10^{-3}\). On the standard double-well potential \(V(x)=(x^2-1)^2\), the test MSE was \(7.88\times10^{-3}\). Expressed in RMSE, those errors were \(5.81\times10^{-2}\) on GRF test data and \(8.89\times10^{-2}\) on the double-well test, which is large for a smooth 1D problem with simple boundary conditions.

That failure mode was useful. It showed that the architecture could fit the training distribution, but the representation and data design were not strong enough to produce reliable operator generalization. Instead of just scaling the same setup, I moved to a more structured parametric family where model choices could be tested more cleanly.

Parametric Potential Family

The refined experiment uses a three-parameter family of potentials:

\[ V(x)=(x^2-1)^2+p_0\exp\left(-\frac{(x-p_1)^2}{2p_2^2}\right), \]

with parameter ranges

  • \(p_0\in[0,5]\)
  • \(p_1\in[-0.5,0.5]\)
  • \(p_2\in[0.25,0.7]\)

This reformulation turns operator learning into a problem with a known latent structure. It keeps the target physics nontrivial while making it easier to reason about coverage, normalization, and out-of-sample generalization across unseen potentials.

Dataset Design

I sample \(N_u=120\) parameter triplets uniformly from the allowed parameter ranges. For each potential, I draw \(N_{\mathrm{eval}}=100\) query locations \(y\sim\mathcal{U}([-1,1])\), compute the corresponding spectral committor, and construct tuples of the form

\[ [p_0,p_1,p_2,y,q(y)]. \]

This yields 12,000 total samples. The critical design choice is that the train/test split is performed by parameter triplet rather than by individual tuple. That means the model is evaluated on entirely unseen potentials, not just unseen query points from familiar parameter settings. This is a stronger and more realistic generalization test for operator learning.

Architecture

The final model is a DeepONet with branch-trunk decomposition:

\[ q_{\theta}(y)=\sigma\big(\langle b,t\rangle+c\big), \qquad b=\mathrm{BranchNN}(p_0,p_1,p_2)\in\mathbb{R}^{m}, \qquad t=\mathrm{TrunkNN}(y)\in\mathbb{R}^{m}, \]

with latent dimension \(m=64\) and sigmoid head \(\sigma(z)=1/(1+e^{-z})\). Both the branch network and trunk network are two-layer MLPs with width 128 and \(\tanh\) activations.

The architecture changes that mattered most were:

  • Parameter normalization for balanced branch-network inputs: \(p_0\mapsto p_0/5\), \(p_1\mapsto (p_1+0.5)\), \(p_2\mapsto (p_2-0.25)/0.45\).
  • Fourier features for the query coordinate: \([y,\sin(\pi y),\cos(\pi y),\sin(2\pi y),\cos(2\pi y),\sin(3\pi y),\cos(3\pi y)]\).
  • A soft boundary penalty \(\lambda_{\mathrm{bc}}\big[(q_{\theta}(-1))^2+(q_{\theta}(1)-1)^2\big]\) with \(\lambda_{\mathrm{bc}}=10^{-2}\).

Together, these changes make the model easier to optimize and better aligned with the structure of the target function class. The Fourier features are especially useful because the committor is smooth but not purely linear in space, so the trunk network benefits from a richer basis than raw \(y\) alone.

Training Setup

The model is trained with Adam using learning rate \(10^{-3}\), batch size 512, and a maximum of 300 epochs. Early stopping is selected based on test RMSE. This is a relatively compact training setup, but it is enough to make the improvement attributable to architecture and representation choices rather than brute-force scaling.

The saved training curve confirms steady optimization. By epoch 1 the train/test RMSE are approximately \(9.90\times10^{-2}\) and \(1.09\times10^{-1}\), and by epoch 270 they reach \(3.92\times10^{-3}\) and \(6.68\times10^{-3}\), respectively. The small train-test gap indicates that the refined DeepONet is not merely memorizing the parameterized training set.

Results

The refined model materially improves over both the original GRF-based DeepONet and the simpler parametric baseline. The test RMSE of \(6.68\times10^{-3}\) is over an order of magnitude smaller than the original GRF-to-committor generalization error of roughly \(8.9\times10^{-2}\). That makes this version of the project much more convincing as an operator-learning result rather than just a training demonstration.

Boundary Behavior and Modeling Tradeoffs

The soft boundary penalty improves endpoint behavior but does not guarantee exact satisfaction of the boundary conditions. The reported value \(q_{\theta}(-1)\approx 0.0099\) is small but not exact. This is an honest limitation of the sigmoid-head formulation. A more principled hard-constraint ansatz is

\[ q_{\theta}(y)=\frac{1+y}{2}+(1-y^2)h_{\theta}(p_0,p_1,p_2,y), \]

which enforces \(q(-1)=0\) and \(q(1)=1\) by construction. I call this out because it clarifies the next step technically: the current model performs well, but there is still room to make the inductive bias more physically faithful.

Takeaways

The main lesson from this project is that DeepONet performance depends strongly on how the operator-learning problem is posed. Once the dataset is rebuilt around unseen parameter triplets and the model is supported by normalization, Fourier features, and a boundary penalty, the method becomes much more reliable and the generalization error drops substantially.

The final result is therefore not just a trained network, but a clearer understanding of which modeling choices make the architecture work on this committor problem and where its remaining limitations still lie, especially in exact boundary enforcement.
