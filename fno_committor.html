Fourier Neural Operator for Committor Problems

This project investigates Fourier Neural Operators for learning committor operators in one dimension. Unlike the DeepONet page, which focuses on pointwise query prediction from a low-dimensional parameter family, this model is built to process full discretized functions and output full discretized committor solutions. That makes it closer to a true function-to-function operator-learning pipeline.

Main role: build and benchmark a Fourier Neural Operator that maps full potential-derivative fields to full committor functions, reaching test RMSE \(4.77\times10^{-3}\) on the parametric dataset and \(1.17\times10^{-3}\) on the Gaussian-random-field dataset.

Operator-Learning Setup

The underlying committor equation is the same one-dimensional boundary-value problem:

\[ q''(x)-\beta V'(x)q'(x)=0, \qquad q(-1)=0,\; q(1)=1, \qquad \beta=1. \]

The key modeling difference is the target mapping. Here the model learns the operator

\[ V'(x)\mapsto q(x) \]

on a fixed discretized grid, so both input and output are entire functions rather than individual point evaluations. This is a setting where FNO is a natural fit because the task has smooth global structure and long-range dependencies across the domain.

Datasets

I build two separate datasets, each with 600 samples discretized on \(N_x=256\) spatial points and split 80/20 into training and test sets:

  • A parametric dataset from the same three-parameter double-well-plus-bump family used in the DeepONet task.
  • A stochastic dataset sampled from a Gaussian random field ensemble.

This dual-dataset setup matters because it evaluates the model in two very different regimes. The parametric dataset tests interpolation and extrapolation across a structured low-dimensional family, while the GRF dataset tests whether the operator can generalize across a broader random function class.

Input Representation

The input representation is one of the strongest parts of the project. For the parametric case, each sample concatenates multiple channels:

  • the raw derivative \(V'(x)\)
  • a normalized copy \(f_{\mathrm{norm}}(x)\)
  • the spatial coordinate \(x\)
  • per-sample mean and standard-deviation channels

This keeps absolute scale information while still stabilizing optimization. For the GRF dataset, I retain the raw derivative, coordinate, and mean/std channels, but intentionally omit the z-scored copy. That design decision avoids washing out the stochastic amplitude information that may be informative for the operator.

That kind of feature design is easy to overlook, but it is a large part of why the final model works well. The architecture is important, but the representation is doing real work here by exposing both global scale and local structure to the network.

FNO Architecture

The final FNO uses width 96, 48 retained Fourier modes, and 4 Fourier layers. Each spectral block applies a Fourier transform, learns a low-mode transformation, and maps back to physical space:

\[ \mathcal{F}^{-1}\big(R_{\mathrm{modes}}\cdot \mathcal{F}(x)\big), \]

where \(\mathcal{F}\) is the Fourier transform and \(R_{\mathrm{modes}}\) is the learned operator in frequency space. Pointwise convolutions complement the spectral blocks, GELU activations are used in the lifted representation, and a sigmoid output head keeps predictions in the physically meaningful interval \([0,1]\).

The architecture is well matched to the task because the committor is a smooth global object. Local pointwise models can fit it, but spectral layers are better aligned with the fact that the solution depends on global characteristics of the potential, not just local neighborhoods.

Training Strategy

The FNO is trained with Adam using learning rate \(10^{-3}\), weight decay \(10^{-6}\), and a maximum of 350 epochs with early stopping. A soft boundary-condition penalty is applied at every batch:

\[ \lambda_{\mathrm{bc}}\big[(q(-1))^2+(q(1)-1)^2\big], \qquad \lambda_{\mathrm{bc}}=10^{-2}. \]

This setup keeps the optimization stable while preserving flexibility in the interior of the domain. As with the DeepONet project, the boundary handling is soft rather than exact, but it is sufficient to keep the predictions physically consistent while avoiding a more restrictive hard-constraint parameterization.

Second, on the more difficult stochastic GRF operator, the FNO reaches \(1.17\times10^{-3}\) test RMSE, which is dramatically better than the earlier DeepONet behavior in the original GRF-to-committor setting.

Why FNO Works Better Here

The clearest distinction between the models is how they process information. DeepONet decomposes the prediction into a branch encoding of the input and a trunk encoding of the query location, then combines them pointwise. That is flexible and useful for arbitrary query evaluation, but it can struggle to capture the full global structure of the operator when the relationship between input and output is better viewed as a field transformation.

The FNO, by contrast, operates directly on discretized functions. Its spectral convolutions expose long-range structure naturally, and its computational profile is attractive for grid-based operator learning. The project report highlights this directly: the Fourier representation is well matched to smooth function spaces with global dependencies, and the empirical results back that up.

Why This Is One of the Strongest Projects

This page is strong because it combines model design, feature engineering, benchmark construction, and comparative evaluation in a way that is easy to defend technically. It is not just an implementation of a published architecture. The important work is in adapting the FNO to this committor setting, choosing channels that preserve amplitude information, benchmarking on both structured and stochastic datasets, and comparing against a serious DeepONet baseline rather than presenting the model in isolation.

The outcome is also unusually clean. The final train/test gaps are small, the operator generalization is strong, and the comparison tells a coherent story: when the task is full-function-to-full-function learning with smooth global structure, the spectral operator model is the better inductive bias.
